# C1-revision-protocol.md 复盘与 SKILL 修订协议

> 流程类 SKILL,不参与图片生成。
> 仅在用户明确说"复盘"或"改SKILL"时触发,其他任何时候保持沉默。

---

## 一、触发条件(强制)

本 SKILL **只在以下两种情况触发**:

1. 用户消息中出现"复盘"二字
2. 用户消息中出现"改 SKILL" / "改skill" / "修订 SKILL" / "更新 SKILL" 等同义表达

**禁止自动触发的情况**:
- 每次迭代结束后不主动复盘
- 每次对话结束不主动复盘
- 用户只说"总结一下"不触发(总结 ≠ 复盘)
- 用户只说"这次解决了啥"不触发本 SKILL,只做简短描述

---

## 二、触发后的执行流程(强制顺序)

```
Step 1:简短复盘(不超过 5 条要点)
Step 2:定位需要修订的 SKILL 文件和段落
Step 3:展示提案(diff + 完整段落,双格式)
Step 4:等待用户确认
Step 5:仅在用户说"确认"后才动手改 SKILL 文件
```

---

## Step 1:简短复盘

- 用 3-5 条要点描述本次迭代解决了什么
- 每条不超过 2 句话
- 禁止长篇大论
- 禁止把每一轮迭代都列出来,只列最终沉淀下来的结论

格式参考:
```markdown
# 这次解决了 N 件事
1. [一句话结论] —— [一句话原理]
2. [一句话结论] —— [一句话原理]
...
```

---

## Step 2:定位修订点

对每一条复盘结论,判断它该写进哪个文件、哪个段落。

**判断原则**:
- 框架级问题(影响整体流程) → SKILL.md / M1 / M2
- 风格级问题(影响视觉一致性) → I1 / D2
- 心智级问题(影响 9 图组合) → I2
- 写法级问题(影响单图描述) → G1 / S1 / P1
- 品类级问题(只影响某一类产品) → B1 / B2 / 未来的 C 系列
- 流程级问题(影响协作方式) → C1 或新建 C 系列文件

如果一条结论涉及多个文件,全部列出。
如果一条结论现有文件无法承载,提议新建文件。

**输出格式(强制)**:

Step 2 的输出必须是表格,不允许散文或列表。

| # | 结论 | 落点文件 |
|---|---|---|
| 1 | [一句话结论] | [文件名 + 段落] |
| 2 | [一句话结论] | [文件名 + 段落] |

如果上下文遗留了未落地的旧结论(用户上一轮提过但还没写进 SKILL),用追加表一并列出,与主表之间用一句话分隔(如"外加上一轮提的 N 条没落地:"):

| # | 结论 | 落点文件 |
|---|---|---|
| N+1 | [遗留结论] | [文件名] |

表格之后用一句话询问用户哪几条进 Step 3,不超过一行。

---

## Step 3:展示提案(双格式,强制)

每一处修订必须**同时**给出两种格式,顺序固定:

### 3.1 diff 格式(看清改动)

```markdown
## 改动 N:[一句话描述]

**位置**:[文件名] §[章节号] [章节标题]

**删除**:
> [原文,逐行引用]

**替换为**:
> [新内容,逐行引用]
```

新增内容用:
```markdown
**新增**(在 §X.Y 之后):
> [新内容]
```

### 3.2 完整段落(改后效果)

紧接 diff 之后给出修订后该段落的完整版本,让用户可以一眼看到改完之后这段读起来是什么样。

格式:
```markdown
**改后完整段落**:
\`\`\`markdown
[完整段落内容]
\`\`\`
```

---

## Step 4:等待确认

提案展示完毕后,**必须停下**,用一句话明确询问:

> 以上是修订提案,确认后我才会动手改 SKILL 文件。你要全部采纳、部分采纳、还是修改后再议?

**禁止行为**:
- 禁止在用户没说"确认"之前动手改任何 SKILL 文件
- 禁止默认用户的沉默等于同意
- 禁止把"听起来不错"等模糊回应当作确认
- 禁止重复追问(用户没回应就等着)

---

## Step 5:执行修订(仅在确认后)

用户的确认信号包括但不限于:
- "确认"
- "改吧"
- "可以,动手"
- "全部采纳"
- "采纳第 1、3 条"(部分采纳)

收到确认后:
1. 按提案逐条修改文件
2. 如果是部分采纳,只改用户点名的条目,其他不动
3. 改完后给一份"修订摘要"(改了哪些文件、哪些段落、改了多少处),不再重复展示完整内容
4. 询问是否需要查看修订后的文件以供核对

---

## 三、禁止事项

- 禁止本 SKILL 自动触发(只在用户明确说"复盘"或"改SKILL"时启用)
- 禁止跳过 Step 3 直接改文件
- 禁止 Step 3 只给 diff 不给完整段落,或只给完整段落不给 diff
- 禁止把用户的模糊回应当作确认
- 禁止一次提案里混入未经讨论的"顺手改进"
- 禁止修改超出本次复盘结论范围的内容

---

## 四、与其他 SKILL 的接口

- 本 SKILL 不影响任何图片生成流程
- 本 SKILL 不被 M1 / M2 调用
- 本 SKILL 仅由用户主动触发
- 本 SKILL 修改的文件可能包括所有其他 SKILL 文件,但每次修改必须经过 Step 4 确认

---

## 五、修订记录

v1.0 首版。动因:某次主图迭代中,用户希望把单次对话沉淀的经验写回 SKILL,但又不希望 AI 每次都自动改文件、不希望长篇大论的复盘、希望明确看到改了什么再确认。本协议为这一需求的固化方案。

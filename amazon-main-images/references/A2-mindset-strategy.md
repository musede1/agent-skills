# A2-mindset-strategy.md 心智组合方案推导引擎

> 基于产品类型+买家认知摘要,动态推导 3 个候选心智组合方案。
> 由 M1-positioning.md 的 Step 5 自动调用。

---

## 核心概念

**主图没有叙事,没有顺序,没有故事。**

主图是九宫格平铺的,买家随机点击,不一定按 Image_1→Image_9 顺序看。
所以"开局/收尾/路径/叙事"这套视频术语在主图里**完全不成立**。

主图真正在做的是 **心智组合**:
9 张图共同传达一组买家心智,不同产品的最优心智组合不同。
9 张图就是这组心智的物理载体。

A2 决定的是:**哪几张图承载哪些心智,几张承载这个,几张承载那个**。

---

## 唯一的"位置意义"

9 张图里只有两个位置有特殊意义:

| 位置 | 意义 |
|---|---|
| Image_1 | 强心智位(九宫格左上角,买家第一眼,缩略图) |
| Image_2 | 尺寸图硬锁 |

其余 Image_3 ~ Image_9 **没有顺序意义**,只有职责类型的权重分布意义。

---

## 铁律

1. Image_2 = 尺寸图(硬锁,所有方案不可移位)— 执行 H1
2. Image_1 = 强心智位(承载方案的核心心智职责)
3. 3 个候选方案的"主导心智"必须各不相同
4. 3 个候选方案的"职责权重分布"必须有差异
5. 方案只写"职责标签 + 权重分布",**禁止写具体场景/道具/画面/物件**
6. 全片人物交互图职责数执行主 SKILL §二 H5(≥2)
7. 同一种职责在同一方案中只出现一次 — 执行 H2
8. **禁止使用"路径/叙事/开局/收尾/故事"等视频术语**
9. 场景组职责数 / 全景数执行主 SKILL §二 H3 / H4

## 推导执行顺序

```
Step 1:读取阶段1的买家认知摘要
Step 2:识别买家决策核心
Step 3:从职责池(D1)中选取 9 个职责
  · 执行主 SKILL §二 H1-H10 全部硬约束
  · 默认场景+置景混搭
Step 4:为 3 个方案分别确定主导心智 + 辅助心智 + 职责权重
Step 5:为每个方案确定 Image_1 强位职责
Step 6:输出 3 个候选方案
Step 7:输出后自检(详见末尾)
```

## 买家决策核心 → 心智方向参考

| 决策核心信号 | 推荐主导心智 | 职责权重特征 |
|---|---|---|
| 好评高频"工艺/做工/手工" | 工艺信任 | 场景图焦点与侧光承载工艺信号,置景组 ≥2 张突出材质氛围 |
| 使用场景"送礼"占比≥50% | 礼品转化 | 礼品场景+礼品递送+礼品感场景 |
| 关键词高频"art / sculpture" | 艺术品质感 | 置景组多 + 场景图聚焦光线承载艺术气质 |
| 差评高频"和家里不搭" | 居家代入 | 场景组多(3-4张) |
| 题材强指向节庆 | 节庆氛围 | 节庆场景 + 礼品场景 |
| A类带功能 | 实用证明 | 功能演示职责多 |
| 差评高频"尺寸不符" | 任何方向 + 尺度强化 | 多张人物交互 |

## 候选方案输出格式

```
【候选心智组合方案】

方案A [名称]
  主导心智:[1-2个心智,如"礼品转化"+"轻奢质感"]
  辅助心智:[1-2个心智]
  职责权重:场景X张 / 置景X张 / 工艺X张 / 信任X张(含尺寸图1张)/ 交互X张(类型1:Y 类型2:Z 类型3:W)
  Image_1 强位职责:[职责名称]
  Image_2:尺寸图(硬锁)
  其余7张职责清单(无顺序意义):
    [职责] / [职责] / [职责] / [职责] / [职责] / [职责] / [职责]
  匹配度:⭐⭐⭐⭐⭐

方案B [名称]
  ...

方案C [名称]
  ...
```

注意:
- "其余7张职责清单"不是按顺序排的,是一个集合,没有先后
- 具体哪个职责落到 Image_3 还是 Image_7,由阶段2的执行决定
- 交互图必须按 D1 §二 的 3 种类型拆分注明(类型1:Y 类型2:Z 类型3:W),
  且 Y+Z+W = 交互X张。三种类型的启用规则见 D1 §二与 §3.3

## 输出后自检(强制)

每个方案输出后,执行者必须自检:

```
1. 是否使用了"路径/叙事/开局/收尾/故事/讲述"等视频术语?
   → 出现即违规,删除并重写

2. 是否出现了具体物件名(蝴蝶结/玫瑰/木桌/咖啡杯等)?
   → 出现即违规,只能保留职责标签

3. 是否出现了具体场景描述(轻奢空间/温柔场景/明亮卧室)?
   → 出现即违规,这是阶段2该决定的事

4. Image_2 是不是尺寸图?
   → 不是即重写

5. 是否至少有1个交互组职责?
   → 没有即重写

6. 3个方案的主导心智是否互不相同?
   → 重复即重写
```

任意一项不通过 → 重写该方案。

---

## 禁止事项

- 禁止输出选择表给用户
- 禁止等待用户选择
- 禁止锁定最终方案
- 禁止 3 个方案的主导心智相同
- 禁止职责标签写成具体画面/物件/场景描述
- 禁止 Image_2 不是尺寸图
- 禁止全片无人物交互职责
- **禁止使用"路径/叙事/开局/收尾/故事"等视频术语**
- 禁止把数据百分比机械映射为职责数量
- 禁止给"其余7张"写顺序

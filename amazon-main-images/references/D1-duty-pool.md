# D1-duty-pool.md 五组职责池(家居摆件通用版)

> 职责池只定"做什么",不定"用什么"。

---

## 一、核心原则

1. 职责池只列职责名称 + 任务说明,不列具体物件清单
2. 具体场景、道具、画面由阶段2根据风格包动态决定
3. 职责池禁止写死视觉细节

---

## 二、五组职责池

### 场景组

| 职责 | 任务说明 |
|---|---|
| 主场景陈列 | 在产品最自然的核心居家空间陈列,作定妆图 |
| 次场景陈列 | 在另一个不同空间陈列,证明百搭性 |
| 礼品感场景 | 在带送礼语境的场景陈列,激发"送给谁"代入 |
| 节庆场景 | 节庆氛围场景,激发应季冲动(限节庆类产品) |

### 置景组

| 职责 | 任务说明 |
|---|---|
| 借景型置景 | 借窗帘/门框/桌面/墙角等环境元素做光与氛围,不交代具体房间 |
| 材质背景置景 | 以某种材质平面(布料/木板/陶板)作背景 |
| 自然元素置景 | 以自然物(干花/石头/苔藓)做氛围陪衬 |

### 信任组

| 职责 | 任务说明 |
|---|---|
| 尺寸图(Image_2 硬锁) | 产品正侧面 + 尺度参照物 + 双单位文字标注 |
| 尺度参照 | 真实场景中通过参照物展示大小(无文字) |

### 交互组

**交互图定义**:画面中含人(手/双手/半身/含脸部)的图,都归入本组。
背景可以是真实空间也可以是虚化背景,不影响类别判定。

按"人物出镜程度"和"背景处理"分为三种类型,每种承担独立任务,
都合法,按产品心智选用。

---

#### 类型 1 · 人物场景交互(有脸 + 真实空间)

画面含人物半身或以上,允许出现脸部(通常是柔和微笑或温柔凝视),
产品 + 人 + 真实居家空间同框。这是主图里最强的情感转化武器。

| 职责 | 任务说明 |
|---|---|
| 人物凝视 | 人物半身 + 温柔注视产品的瞬间 |
| 人物轻触 | 人物半身 + 单手手指轻触产品顶/侧 |
| 人物双手捧 | 人物半身 + 双手捧持产品展示给镜头 |
| 人物携带 | 人物半身 + 双手捧出或抱在怀中 |

**启用门槛(重要)**:类型 1 仅在 I2 心智方案锁定为以下之一时允许启用:
- 情感心智(爱情 / 家庭 / 纪念 / 母亲节 / 陪伴)
- 礼品转化心智(高端礼物 / 送人场景)
- 感官心智(香薰 / 蜡烛 / 疗愈)

以下心智禁止启用类型 1(防止滥用人脸造成尴尬):
- 实用证明(容器/收纳/功能摆件)
- 艺术品质感(纯装饰/工艺收藏)
- 工艺信任(手工/材质展示)
- 居家代入(普通家居摆件)

违反启用门槛 → M1 Step 5 候选方案推导时直接禁用该类型。

**类型 1 的双重计数**:类型 1 画面中既有完整真实空间又有人物,
承担"带人的场景图"双重身份,**同时计入 H3 场景组与 H5 交互组**。
这是 H3/H5 计数中唯一允许双重计数的情况。

---

#### 类型 2 · 手部场景交互(无脸 + 真实空间)

画面只含手或前臂,无脸部出现,产品 + 手 + 真实居家空间同框。
任务是证明"在这个空间里怎么用"。

| 职责 | 任务说明 |
|---|---|
| 场景内单手放入 | 真实空间 + 单手放入动作(放糖/放钥匙/放物) |
| 场景内单手取出 | 真实空间 + 单手取出动作(取钥匙/取糖) |
| 场景内单手轻触 | 真实空间 + 单手两指轻触产品顶/侧 |
| 场景内插花瞬间 | 真实空间 + 单手持一支花入瓶 |
| 场景内注水瞬间 | 真实空间 + 单手持壶注水入瓶 |
| 场景内双手入瓶 | 真实空间 + 双手捧花束入瓶 |
| 场景内双手归位 | 真实空间 + 双手归置产品 |

**默认规则**:所有单手弱动作(轻触/放入/取出/插花/注水)默认归入类型 2,
即便动作发生在桌面上也要把完整空间带进画面。

---

#### 类型 3 · 手部特写交互(无脸 + 虚化背景)

画面只含手,背景含蓄虚化,手与产品占画面主导。任务是"手与产品的
直接对话",证明"掌中体量 / 材质触感 / 礼品仪式"。

| 职责 | 任务说明 |
|---|---|
| 双手捧持 | 双手从两侧捧住,展示体量与质感 |
| 双手捧出 | 双手向镜头递出 |
| 礼品递送 | 双手捧产品作为礼品递出 |
| 转瓶展示 | 单手转动产品中的瞬间(纯工艺证明,有明确独家理由时启用) |
| 瓶口俯览 | 单手扶持俯拍(纯工艺证明,有明确独家理由时启用) |

**默认规则**:双手强对话动作(捧持/捧出/递送)默认归入类型 3,
干净背景才能突出动作本身的仪式感。

---

### 三种类型速查表

| 维度 | 类型 1 | 类型 2 | 类型 3 |
|---|---|---|---|
| 人物范围 | 半身+脸 | 只有手 | 只有手 |
| 背景 | 真实完整空间 | 真实完整空间 | 虚化含蓄 |
| 画面主角 | 人+产品的情感瞬间 | 动作+空间 | 手与产品对话 |
| 适用心智 | 情感/礼品/感官 | 实用/使用证明 | 工艺/体量/质感 |
| 光线组 | A+B+C+D | A+B+C+D | A+B+D(C 可选) |
| 计入 H3 场景组 | **是**(双重计数) | 否 | 否 |
| 计入 H5 交互组 | 是 | 是 | 是 |

## 三、职责选择规则

### 3.1 硬性规则

执行主 SKILL §二 全局硬约束清单 H1 / H2 / H3 / H4 / H5(Image_2 硬锁 / 职责去重 / 场景 ≥4 / 全景 ≥3 / 交互 ≥2)。
默认场景+置景混搭。

### 3.2 软性建议(由心智组合方案决定)
- 场景组下限由 H3 锁定,上限由 H5 + H1 反推
- 置景组数量 = 9 − 场景组 − 信任组 − 交互组
- 信任组 1 张(Image_2 尺寸图必有)
- 交互组下限由 H5 锁定

### 3.3 交互图三种类型的数量规则

- **H5 保持"交互组 ≥2"**(总数下限)
- **类型 1 数量**:
  - 情感/礼品/感官心智 → 允许 1-2 张(2 张上限)
  - 其他心智 → 禁止启用(启用门槛见 §二 类型 1)
  - 类型 1 **同时计入 H3 场景组与 H5 交互组**(双重计数)
- **类型 2 数量**:无上限,但同一动作不可重复(H2 约束)
- **类型 3 数量**:每片最多 1 张(防止全片变成虚化特写堆)
- **三种类型允许自由混搭**,全片 ≥2 张即满足 H5

### 3.4 单手弱动作 vs 双手强动作的默认分配

| 动作 | 默认类型 |
|---|---|
| 单手轻触 / 单手放入 / 单手取出 / 单手插花 / 单手注水 | 类型 2(场景支撑) |
| 双手入瓶 / 双手归位(动作发生在真实台面)| 类型 2(场景支撑) |
| 双手捧持 / 双手捧出 / 礼品递送 | 类型 3(干净背景突出仪式) |
| 任何含人物半身或脸部的动作 | 类型 1(情感心智驱动) |

判断核心:动作本身信息量弱 → 需要空间撑(类型 2);
动作本身是强仪式 → 需要干净背景突出(类型 3);
产品需要情感锚点 → 必须人脸(类型 1)。

## 六、特殊职责处理

### 6.1 节庆场景
仅当产品题材本身明确为节庆产品时启用。

### 6.2 尺度参照 vs 尺度交互
- 尺度参照:用家居物做参照,无人物
- 尺度交互:用手部做参照,有人物
- 全片只能选一个,不可同时

## 七、A 类功能摆件的装/空分配原则

> A 类产品(容器/收纳/功能性摆件)的核心卖点是"能装东西"。
> 9 张图里不是每张都装满,也不是每张都空,而是按图的任务分配状态。

### 7.1 核心逻辑

**图的任务决定装不装**,不是"为了装而装",也不是"为了美而空"。

### 7.2 按职责分类的装/空分配

| 职责类型 | 状态 | 理由 |
|---|---|---|
| **场景组**(主场景/次场景/礼品场景) | **装满** | 任务是"证明能用"+"融入家",必须展示真实使用状态 |
| **交互组**(手部演示) | **半满 → 正在加** | 任务是"证明怎么用",必须呈现动作的中间态(空无法演示,满看不出动作) |
| **置景组**(借景/材质背景/自然元素) | **空** | 任务是"证明产品自身质感+氛围",装物会抢产品本体戏 |
| **信任组**(尺寸图) | **空** | 规格图必须展示产品原貌 |

### 7.3 典型 9 图分配节奏

```
装满 3 张(全部场景组) + 半满 1 张(交互组) + 空 5 张(置景/规格)
```

这个比例是 A 类主图的推荐默认节奏。场景组图数增减时,装满图数随之调整。

### 7.4 装物选择原则

- 装物内容必须与**归属空间的使用场景**匹配(梳妆台装珠宝/玄关装钥匙零钱/书桌装文具)
- 装物必须"接近装满"或"饱满溢出",禁止只放 1-2 件小物"意思一下"
- 装物色系必须与风格包点缀色兼容,不得出现风格包禁用色
- 装物堆叠状态应自然无序,禁止摆放得过于整齐

### 7.5 半满状态(交互图专用)

- 容器里已有一部分物品,手部正在添加新物品
- 已有物品必须"可见但未满",为手部动作留出视觉空间
- 手部添加的物品类型应与已有物品呼应(同类或相关)

### 7.6 prompt 写法锚定词

- **装满**:`generously filled with [具体内容] piled naturally together almost reaching the rim`
- **半满**:`already holding a small pile of [已有内容] / lowering an additional [新物品] toward [容器口]`
- **空**:`with the [容器部位] empty` 或直接不提容器内部状态

### 7.7 铁律

- 全片装满图 ≥ 2 张(否则买家看不到"装物"的核心转化证据)
- 全片空图 ≥ 3 张(否则买家看不到产品本体造型)
- 交互组必有 1 张"正在添加"的半满状态图
- Image_2 复合尺寸图的细节 panel 若拍容器内部,**必须空**(这是展示容量的唯一机会)

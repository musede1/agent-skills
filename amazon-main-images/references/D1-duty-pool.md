# D1-duty-pool.md 五组职责池(家居摆件通用版)

> 职责池只定"做什么",不定"用什么"。

---

## 一、核心原则

1. 职责池只列职责名称 + 任务说明,不列具体物件清单
2. 具体场景、道具、画面由阶段2根据风格包动态决定
3. 职责池禁止写死视觉细节

---

## 二、五组职责池

### 场景组

| 职责 | 任务说明 |
|---|---|
| 主场景陈列 | 在产品最自然的核心居家空间陈列,作定妆图 |
| 次场景陈列 | 在另一个不同空间陈列,证明百搭性 |
| 礼品感场景 | 在带送礼语境的场景陈列,激发"送给谁"代入 |
| 节庆场景 | 节庆氛围场景,激发应季冲动(限节庆类产品) |

### 置景组

| 职责 | 任务说明 |
|---|---|
| 借景型置景 | 借窗帘/门框/桌面/墙角等环境元素做光与氛围,不交代具体房间 |
| 材质背景置景 | 以某种材质平面(布料/木板/陶板)作背景 |
| 自然元素置景 | 以自然物(干花/石头/苔藓)做氛围陪衬 |

### 工艺组

| 职责 | 任务说明 |
|---|---|
| 材质特写 | 镜头聚焦产品材质质感 |
| 工艺细节 | 镜头聚焦工艺细节(雕刻/接缝/纹样) |
| 造型多角度 | 从不同于正面的角度拍摄,展示造型完整性 |
| 色彩/釉面特写 | 镜头聚焦色彩或釉面变化 |

### 信任组

| 职责 | 任务说明 |
|---|---|
| 尺寸图(Image_2 硬锁) | 产品正侧面 + 尺度参照物 + 双单位文字标注 |
| 尺度参照 | 真实场景中通过参照物展示大小(无文字) |

### 交互组

| 职责 | 任务说明 |
|---|---|
| 尺度交互 | 手部持握产品,直观感知大小 |
| 礼品递送交互 | 双手捧出/递出产品,强化送礼属性 |
| 材质触摸交互 | 手指轻抚产品,证明质感真实可触 |

---

## 三、职责选择规则

### 3.1 硬性规则
- Image_2 必为"尺寸图"职责
- 9 张图至少包含 1 个交互组职责
- 9 张图职责互不重复(同一种职责只出现一次)
- 默认场景+置景混搭

### 3.2 软性建议(由心智组合方案决定)
- 场景组通常 3-4 张
- 置景组通常 3-5 张
- 工艺组通常 1-3 张
- 信任组 1-2 张(尺寸图必有)
- 交互组 1-2 张

---

## 四、造型多角度的"角度选择"必须服从叙事面

| 叙事面 | 允许角度 | 禁止角度 |
|---|---|---|
| 全方位型(360°都有内容) | 后角 / 侧后 / 三分之四后 / 俯角 / 任何角度 | 无 |
| 仅正面型(浮雕/正面叙事) | 正面 ±15° / 微仰 / 微俯 / 局部特写 | 后角 / 纯侧面 / 三分之四后 |

判断标准:**如果产品的核心识别特征只在某一面,任何看不到这一面的角度都是禁止的**。
浮雕型/相框型/带画面元素的正面叙事产品,造型多角度只能从正面附近变化,不能绕到后面。

---

## 五、职责与"是否含人物"对照

| 职责 | 是否含人物 |
|---|---|
| 场景组 | 默认无,礼品感场景可选有 |
| 置景组 | 默认无 |
| 工艺组 | 默认无(干净特写) |
| 尺寸图 | 无 |
| 尺度参照 | 默认无,可选 |
| 交互组 | 必有手部 |

---

## 六、特殊职责处理

### 6.1 节庆场景
仅当产品题材本身明确为节庆产品时启用。

### 6.2 尺度参照 vs 尺度交互
- 尺度参照:用家居物做参照,无人物
- 尺度交互:用手部做参照,有人物
- 全片只能选一个,不可同时

---

## 七、扩展机制

未来若遇到当前五组无法覆盖的产品类型(发光摆件/动态摆件/套组产品),
通过 C 系列品类专项文件扩展,不修改本文件。

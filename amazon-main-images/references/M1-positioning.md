## Step 1:数据解析

如用户提供数据文件 → 执行;两份都未提供 → 跳过,基于产品图自行判断。

### 1.1 评论分析报告
读取5个固定维度,提取信号:

| 维度 | 提取 | 用途 |
|---|---|---|
| 使用场景 | TOP3+占比 | 决定场景图空间方向 |
| 好评 | TOP3+占比 | 决定必保"证明型"职责 |
| 差评 | TOP3+占比 | 决定必保"预防型"职责 |
| 购买动机 | TOP3+占比 | 决定整体视觉质感 |
| 未被满足的需求 | 全部 | 决定差异化机会 |

**数据有效性判断:** 如果数据明显与产品不符(例如其他品类的污染数据),自动判定为无效,只用关键词数据,不在决策中使用。

### 1.2 关键词搜索报告
- 高频词的品类定位 → 风格方向
- 场景词提取 → 直接映射归属空间
- 高搜索量+低排名词 → 差异化机会点

### 1.3 输出《买家认知摘要》
```
核心使用场景TOP3 + 占比
必保卖点TOP3
必消顾虑TOP3
购买动机TOP3
风格认知方向(高频心智词+心智判定)
空间认知方向
差异化机会点
```

## Step 2:产品信号读取

| 信号维度 | 提取内容 |
|---|---|
| 材质 | 玻璃/陶瓷/树脂/金属/木/石 |
| 表面处理 | 哑光/高光/透明/做旧/手绘 |
| 主色 | 冷暖+饱和度 |
| 造型 | 写实/抽象/几何/有机/仿生 |
| 题材 | 海洋/动物/人物/花鸟/节庆/抽象 |
| 工艺特征 | 手工吹制/手雕/手绘等 |

**新增字段:叙事面判定(关键)**

| 类型 | 判定 | 拍摄影响 |
|---|---|---|
| 全方位型 | 360°都有内容(独立体雕塑/瓶罐/动物等) | 允许后角/侧后/俯角等大角度 |
| 仅正面型 | 核心叙事只在正面(浮雕/相框/带画面元素的扁平摆件) | 禁止后角/纯侧面,所有图只能正面±15°/微仰/微俯 |

判断标准:**如果产品的核心识别特征(图案/表情/中心元素)只在某一个面,就是仅正面型。**

### 参考图盘点(强制)

M1 Step 2 结束时必须输出参考图盘点表:

| 角度 | 是否提供 | 来源图 |
|---|---|---|
| 正面/前侧 | 是/否 | — |
| 纯侧面 | 是/否 | — |
| 背面 | 是/否 | — |
| 顶视 | 是/否 | — |
| 底部 | 是/否 | — |
| 功能状态(容器带物/瓶插花等) | 是/否 | — |

盘点结果决定:
- **尺寸图形态选择**:≥3 张 + 工艺为核心卖点 → 允许复合尺寸图;否则纯净尺寸图

盘点结果必须完整传递给阶段2。

### 子 SKILL 激活
- 若产品类型=B(花瓶) → 自动激活 `B2-vase.md`,后续所有步骤叠加 B2 规则
- 若产品类型=A(容器/收纳/功能性摆件) → 自动激活 `B1-container.md`,后续所有步骤叠加 B1 规则

## Step 4:候选风格推导

调用 `I1-style-engine.md`,输出 3 个候选风格 + 匹配度。

铁律:
- 主图风格必须在"明亮安全区"内
- 暗调/戏剧化默认排除,除非题材强指向暗黑
- 禁止在阶段1锁定最终风格

---

## Step 5:候选心智组合方案推导

调用 `I2-mindset-strategy.md`,输出 3 个候选心智组合方案(职责标签 + 权重分布,无顺序)。

禁止在阶段1锁定最终方案。
禁止使用"路径/叙事/开局/收尾/故事"等视频术语。

**若已激活 B2**:3 个候选心智方案必须各带"花瓶呈现模式"标签(纯工艺型/纯生活型/均衡型)。

**若已激活 B1**:3 个候选心智方案必须各带"容器呈现模式"标签(纯工艺型/纯生活型/均衡型)。

**交互图类型约束(全品类强制)**:

3 个候选方案的"职责权重分布"必须显式注明允许的交互图类型(定义见 D1 §二 交互组):

- 主导心智 ∈ {情感 / 礼品 / 感官} → 允许 类型 1 + 类型 2 + 类型 3
- 主导心智 ∈ {实用证明 / 居家代入} → 允许 类型 2 + 类型 3,**禁止类型 1**
- 主导心智 ∈ {艺术品质感 / 工艺信任} → 允许 类型 2 + 类型 3,**禁止类型 1**

违反类型 1 启用门槛 → 该候选方案重写,禁止把人脸塞进非情感类产品。

# B1-container.md A 类功能容器摆件主图专项子 SKILL

> 由 M1 Step 2 识别产品类型=A 时自动激活。
> 主图无叙事、无顺序、9 张平铺,所有规则按"单张图"维度生效。

---

## 一、适用范围

- 适用于所有 A 类产品(容器/收纳/功能性摆件)
- 判断标准:**产品有可以放东西的内部空间**
- 例:catchall bowl、首饰盒、储物罐、托盘、笔筒、嘴袋摆件、钥匙挂、糖果罐等
- 不适用于纯实心装饰摆件(走 C 类常规流程)

---

## 二、核心原则

1. A 类产品的核心卖点是"能装东西",不装东西就是只展示了一半
2. 每张图的"装/空"状态由该图的**任务类型**决定,不是越满越好
3. 装物必须服务"使用故事",不是凭空摆好看
4. 视觉一致性优先于数量 — 碗里的物品必须能被读成"一种东西"

---

## 三、容器呈现模式(I2 推导补充)

> A 类的等价物是:I2 心智方案必须显式声明本片的"容器呈现模式"。

### 三种呈现模式

| 模式 | 装物图占比 | 空图占比 | 适用心智 |
|---|---|---|---|
| **纯生活型** | 多数(5–6张)| 少数(3–4张)| 居家代入 / 礼品转化 / 实用证明 |
| **均衡型** | 中等(3–4张)| 中等(5–6张)| 综合心智(默认推荐) |

### I2 推导铁律(B1 追加)

- I2 输出 3 个候选心智方案时,3 个方案的呈现模式不得全部相同
- 每个方案必须显式标注:`容器呈现模式:[纯生活型/均衡型]`
- 全片至少 1 张装物图(防止全片只见容器不见用法)
- 全片至少 1 张空图(防止全片只见装物不见产品本体)
- 默认推荐"均衡型",除非心智明确为纯实用导购

## 四、容器专属职责(D1 交互组扩展)

> A 类专属交互动作,追加到 D1 交互组,不影响其他品类。

| 职责名称 | 单图呈现 | 手部 | 通常对应模式 |
|---|---|---|---|
| **单手放入** | 单手将物品放入容器的瞬间 | 单手 | 任意 |
| **单手取出** | 单手将物品从容器取出的瞬间 | 单手 | 纯生活型 |
| **双手捧出** | 双手捧着容器向镜头递出 | 双手 | 礼品心智 |
| **双手捧持** | 双手捧住容器展示体量与质感 | 双手 | 任意 |
| **倾倒展示** | 单手倾斜容器,内容物即将倾出的瞬间 | 单手 | 纯生活型(食物类装物) |
| **整理归位** | 单手将容器内物品整理归位的瞬间 | 单手 | 纯生活型 |
| **礼品递送** | 双手捧空容器作为礼品递出 | 双手 | 礼品心智 |

### 职责选用规则

- 全片至少 1 张交互职责(继承 D1 总规则)
- 同一种交互职责全片只用 1 次
- 交互动作必须与该图的装/空状态自洽:
  - 单手放入 → 半满状态(已有同类物品 + 正在加新的)
  - 单手取出 → 半满状态(已有物品 + 正在拿走一件)
  - 倾倒展示 → 满状态(必须装满才能倾倒)
  - 双手捧持 / 捧出 / 礼品递送 → 通常空状态

## 五、装/空分配总表(强制前置执行)

> 必须在 M2 Step 6 职责分配完成之后、Step 8 写图之前完整填写并通过自检。
> 未通过自检禁止开始写任何 Image_N。

### 填写顺序(强制)

```
Step 1:确认每张图的呈现状态(空 / 装物 / 半满)
Step 2:对所有装物图,根据该图职责与场景推导装物类型(少件型/一捧型)
Step 3:对所有装物图,根据空间和使用故事推导具体装物内容
Step 4:填写总表
Step 5:视觉一致性自检
Step 6:全部通过 → 开始写 Image_N
```

### 装/空状态分配逻辑(按职责类型)

| 职责类型 | 状态 | 理由 |
|---|---|---|
| **场景组**(主场景/次场景/礼品场景) | **装物** | 任务是"证明能用"+"融入家",必须展示真实使用状态 |
| **交互组**(手部演示) | **半满 → 正在加/取** | 任务是"证明怎么用",必须呈现动作的中间态 |
| **置景组 - 借景型**(窗边/门框) | **空** | 任务是展示产品釉面/材质在环境光下的质感,装物会遮挡 |
| **置景组 - 材质背景**(亚麻/陶板) | **装物**(氛围型) | 任务是材质对话,装物不影响,反而补足功能心智 |
| **置景组 - 自然元素**(干枝/石头) | **装物**(自然延伸型) | 任务是氛围,装物与置景同源不冲突 |
| **信任组 - 尺寸图** | **空** | 规格图必须展示产品原貌 |

## 六、装物核心原则

### 6.1 视觉一致性(最高原则)

碗/容器里的物品必须能被视觉读成**"一种东西"**。

判断标准:**眯起眼睛看一眼,看到的是"一类东西",还是"几类东西堆在一起"**?
- 看到"一捧 X" → 通过
- 看到"X 和 Y 和 Z" → 不通过

### 6.2 两种装物模式

| 模式 | 适用品类 | 数量 | 视觉效果 |
|---|---|---|---|
| **少件型** | 单件价值物(钥匙/手表/戒指/眼镜/手机) | 2-3 件 | 精选,松散摆放 |
| **一捧型** | 小颗粒物(糖果/松果/水果/坚果/干花瓣/海盐) | 一大捧 | 单一品类,自然堆叠 |

**判断标准**:物品本身体积小且复数存在感强 → 一捧型;物品本身体积大或单件即有存在感 → 少件型。

### 6.3 装物自然量参考表

| 品类 | 自然丰盈量 | 备注 |
|---|---|---|
| 钥匙 | 1-2 串 | 多了像锁匠 |
| 手表 | 1 只 | 1 只就是日用,2 只起就奇怪 |
| 戒指 | 2-4 枚 | 摊开摆放,松散自然 |
| 眼镜 | 1 副 | 折叠或展开均可 |
| 珠宝混搭 | 3-5 件 | 同大类内部混搭(戒指+耳钉+项链) |
| 松果 | 4-6 颗 | 一捧的视觉密度 |
| 糖果 | 20+ 颗 | 满满一碗才是糖果碗 |
| 水果 | 3-5 个 | 几个堆在一起就是果碗 |
| 坚果 | 一捧 | 不可数,自然堆叠 |
| 干花瓣 | 一小把 | 少了寒酸,多了抢戏 |
| 薄荷糖 | 一捧 | 同糖果 |
| 文具(回形针等) | 一小撮 | 桌面收纳逻辑 |

## 七、装物服务"使用故事"

不是"碗能装什么",而是"**这个空间的人会在这个碗里放什么**"。

### 装物必须由空间反推

| 空间 | 真实使用情境 | 装物推荐 |
|---|---|---|
| 玄关控台 | 进门的人放下随身物 | 钥匙 + 手表(出门带的东西)|
| 床头柜 | 睡前摘下的物件 | 戒指 + 眼镜(休息前摘下)|
| 梳妆台 | 化妆时摘下的首饰 | 珠宝(戒指/耳钉/项链)|
| 客厅边几 | 招待客人 | 糖果 / 坚果 / 薄荷糖 |
| 书桌 | 工作物品收纳 | 回形针 / 橡皮筋 / 小文具 |
| 餐桌中央 | 日常果碗 | 水果 |
| 礼品场景 | 收到的礼物 | 珠宝 / 精致小物 |

### 铁律
- 装物必须与空间真实使用逻辑匹配
- 禁止"凭空想放什么好看"
- 禁止把日用品(钥匙/手表)放进礼品场景
- 禁止把珠宝放进客厅边几(应该是糖果)

---

## 八、置景图装物的特殊规则

置景图里的装物**不能装日用品**(钥匙/手表/手机会破坏置景的诗意),只能装**与置景元素同源**的内容物。

### 置景图装物对照表

| 置景类型 | 装物类型 | 装物推荐 |
|---|---|---|
| 借景型(窗边/门框) | **空** | — |
| 材质背景(亚麻+陶板) | 一捧型 / 少件型 | 干花瓣 / 小石子 / 木珠 / 干枝小段 |
| 自然元素(干枝+石头) | 一捧型 | 松果 / 干叶 / 坚果 / 海盐 |

### 铁律

- 置景图装的物品必须与置景元素**同源**(自然 vs 自然 / 静物 vs 静物)
- 禁止日用品出现在置景图里
- 借景型置景一律保持空,不装任何东西

## 九、prompt 锚定句式

### 9.1 少件型(精选 2-3 件同类)

```
holding [具体物品 X] and [具体物品 Y] in a natural everyday arrangement
```

示例:
- `holding a single set of house keys on a brown leather fob and one men's watch with a leather strap in a natural everyday arrangement`

### 9.2 一捧型(单一品类丰盈)

```
filled with a generous handful of [单一品类] naturally piled together as a single unified element
```

示例:
- `filled with a generous handful of small natural pinecones naturally piled together as a single unified element`
- `filled with a generous handful of dried pale rose petals naturally piled together as a single unified element`

### 9.3 半满交互(已有 + 正在加)

```
already holding [已有同类物品] visible inside, a single adult hand 
gently lowering [新的同类物品] toward the [容器口] in a natural placing gesture
```

示例:
- `already holding a single set of house keys on a leather fob visible inside, a single adult hand gently lowering one men's watch toward the bowl in a natural placing gesture`

### 9.4 空状态

直接写 `with the [容器内部] empty`,或不提容器内部状态。

## 十、唯一禁止(铁律)

### 10.1 多类小数量混堆(最高禁止)

**禁止"3 类以上各 1-2 件"的混堆**。这是杂物盒,不是装饰碗。

❌ 错误示例:
- 钥匙 1 串 + 戒指 2 枚 + 墨镜 1 副 + 车遥控 1 个
- 手表 1 只 + 戒指 1 枚 + 老花镜 1 副 + 发卡 1 枚

✅ 正确示例:
- 钥匙 1 串 + 手表 1 只(2 类,出门类同源)
- 戒指 2 枚 + 耳钉 1 对(同属珠宝大类)
- 一捧松果(单类)

### 10.2 类别数判定规则

单图碗内**类别数 ≤ 1**(同大类内部混搭除外):
- 钥匙 + 手表 → 2 件,但同属"出门带的",视觉读作"一类",通过
- 戒指 + 耳钉 + 项链 → 3 件,但全是珠宝,通过
- 钥匙 + 戒指 → 2 件,跨类别(出门 vs 珠宝),不通过
- 松果 6 颗 → 1 类,通过

判断仍然走视觉一致性原则(§6.1):眯眼看是不是一类东西。

### 10.3 其他全局禁止

- 禁止全片没有装物图(看不到功能)
- 禁止全片没有空图(看不到产品本体)
- 禁止 Image_2 尺寸图带装物
- 禁止 Image_2 尺寸图带装物
- 禁止礼品场景装日用品
- 禁止置景图装日用品
- 禁止"为了装而装"(违背使用故事)
- 禁止"为了空而空"(全片置景化,放弃功能心智)

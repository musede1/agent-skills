# 亚马逊主图视频全流程生产线

---

## 【高频漏项强制核查】

> 每个Shot生成前必须过一遍。出现任何一项不达标，禁止继续输出。

### A类 · FI核查（每Shot生成前）
- ❶ **FI件数尽量多放**：尽量接近容量上限，不得每个Shot都明显偏少
- ❷ **至少1个镜头明显少于上限**：全片中至少有1个镜头件数明显低于最大容量
- ❸ **镜头间件数必须有变化**：不得全片每个镜头FI件数相同
- ❹ **同一FI物件全片最多出现2次**：第3次必须强制替换为其他物件，不得以任何理由延续

### A类 · Decor核查（每Shot生成前）
- ❺ **全景/中景强制2–3件**：未达下限禁止输出该Shot，必须先补充道具
- ❻ **近景/中近景至少1–2件**
- ❼ **特写/微距允许0件**：此景别不受数量下限约束

### B类/C类 · Decor核查（每Shot生成前）
- ❽ **全景/中景至少1–3件**：下限为1件，不得全程零道具
- ❾ **近景/中近景至少1–2件**
- ❿ **特写/微距允许0件**

### 全类型 · 运镜核查（全片规划时）
- ⓫ **第二类固定镜头全片≤2次且不得相邻**：超限或相邻必须调整
- ⓬ **全片必须出现1–2次组合运镜**：不得全片无组合运镜
- ⓭ **开场第一个镜头禁止使用组合运镜**
- ⓮ **同一空间场景全片最多出现3次**：超限必须换场景

### A类 · 叙事顺序核查（脚本完成后）
- ⓯ **A类镜头顺序必须符合该产品的真实使用流程，不得倒置动作因果**（具体流程由产品功能决定，例：收纳碗为"放入→归位"；储物盒为"取出→使用→放回"）

## 一、总原则（优先级从高到低）

1. 先定位产品，再选叙事主线，再选场景，再生成视觉参数；禁止先选风格再硬塞场景。
2. 场景由产品自然归属空间决定，不由视觉参数包决定。
3. 视觉参数包只负责色彩/材质/光线，不负责场景/道具数量/构图。
4. 叙事主线由产品类型+转化目标决定，不由模板写死。
5. 规则按层级执行：全局规则 > 产品分类规则 > 场景资产规则 > 提示词写法规则 > 分镜规则 > 后置校验。
6. 默认画面比例固定为16:9，写在正向提示词最后；只有用户明确指定其他比例时才覆盖。
7. 同场景连续镜头必须继承空间与道具关系，但摄影机位必须变化。
8. 禁止前后双台面、中岛分离、纯摆拍空台面、物件围绕镜头刻意排列。

## 二、标准输入格式

```
产品图：[上传图片]
站点：美国 / 欧洲 / 日本
类目：[填写]
材质：[填写]
功能属性：[填写]
尺寸：[填写，如"高16cm"]
时长：[填写，如"28秒"]
比例：16:9（默认）/ 9:16 / 1:1
竞品差异化：[可选，如"同类都是透明玻璃罐，我们是陶瓷+棋格图案"]
视频转化目标：[可选，如"主打密封性和颜值差异化"]
工作流：[不填默认第一版 / 简化版]
```

> `竞品差异化`、`视频转化目标` 和 `工作流` 为可选项。
> `工作流` 未填写时默认输出第一版（完整JSON）。
> `工作流：简化版` 时阶段3输出第二版（全能参考格式，适用于Seedance 2.0一张产品图生成15s完整视频）。

## 三、产品分类（强制前置）

### A类：容器 / 收纳 / 功能性摆件
- 有明确"装/放/收/取/承载"功能（台面落点）
- 允许：Base + Decor + Functional Items + 同类陈设组
- 叙事逻辑：以真实使用流程为骨架，镜头顺序不得违反动作因果

### B类：花瓶类
- 独立大类，不并入A类
- 允许：Base + Decor + 花材规则
- 执行时额外调用 `R05-vase-flower-guide.md`
- 叙事逻辑：由R05叙事锚点池定义，E2动态推导，不预设叙事结构

### C类：纯雕像 / 纯装饰摆件
- 无明确使用动作，以视觉陈列为主（台面落点）
- 允许：Base + Decor
- 叙事逻辑：纯情绪递进，无需动作因果链

### D类：墙饰类
- 以挂墙为主要落点，细分为D-装饰型和D-功能型
- 执行时额外调用 `R08-wall-decor-guide.md`
- **D-装饰型**：视觉陈列为主，无明确使用动作；叙事逻辑：情绪递进，以工艺质感与空间融合为核心
- **D-功能型**：有明确使用动作（挂钟/壁挂花插/置物壁架等）；叙事逻辑：继承A类使用流程逻辑
- 全片必须有至少1个挂墙场景镜头

## 四、三阶段流程路由

### 阶段1：产品定位
执行前必须读取：`P1-positioning.md`
- 输入：产品图 + 标准输入格式
- 执行：读取产品信号 → 输出定位结果（含D类子类判定）→ 动态推导3个候选风格 → 动态推导3个候选叙事主线
- 输出：完整产品定位包（定位结果 + 3个候选风格名称 + 3个候选叙事主线）
- **全程自动执行，无人工介入**
- 阶段1完整输出后，自动进入阶段2

### 阶段2：视频脚本生成
执行前必须读取：`P2-script.md`
- 输入：阶段1完整定位包
- 执行：锁定最终风格 → 动态生成视觉参数包 → 锁定最终叙事主线 → 按主线分配镜头职责 → 生成N镜头×7字段脚本 → 三项校验
- 输出：风格锁定声明 + 视觉参数包 + 叙事主线锁定声明 + N镜头×7字段完整脚本
- **全程自动执行，无人工介入**
- 阶段2完整输出且三项校验通过后，自动进入阶段3

### 阶段3：分镜生成
执行前必须读取：`P3-storyboard.md`
- 输入：阶段1定位包 + 阶段2完整脚本（含视觉参数包）+ 用户工作流声明
- 执行：判断工作流版本 → 按对应格式逐Shot生成
- 输出：
  - **第一版（默认）**：完整分镜JSON + 完成汇总表
  - **第二版（简化版）**：全局声明 + N条Shot提示词（适用于Seedance 2.0全能参考模式）
- **全程自动执行，无人工介入**

## 五、跨阶段传递规则（强制）

1. 阶段1输出的产品类型/尺寸档位/归属空间/排除空间/使用动作/3个候选风格/3个候选叙事主线，必须完整传递给阶段2
2. 阶段2锁定的风格名称、视觉参数包、叙事主线，必须完整传递给阶段3
3. 阶段2每个Shot的时长/画面内容/商业动作/音效，必须完整传递给阶段3
4. 禁止任何阶段跳过上一阶段直接执行
5. 禁止阶段2在候选3个之外另选风格或叙事主线
6. 禁止阶段3自行重新判定风格或场景

## 七、禁止事项（全局）

- 禁止视觉参数包提供场景预设或道具数量限制
- 禁止先选风格再选场景
- 禁止在叙事主线锁定之前填写任何镜头内容
- 禁止在视觉参数包生成之前开始写任何镜头画面内容
- 禁止A类产品镜头顺序违反真实使用流程
- 禁止在动作镜头里滥用同类陈设组抢主角
- 禁止把花瓶并入普通容器类
- 禁止把墙饰并入C类装饰摆件
- 禁止Decor对账未达标就输出Shot内容
- 禁止FI物件超过2次仍继续使用
- 禁止风格候选因"有现成参数文件"而被隐性加权
- 禁止任何阶段有人工介入节点
- 禁止全局规则写死进每个Shot里
- 禁止D类产品全片无挂墙场景镜头
- 禁止D-功能型产品在台面演示功能动作

## 八、最终提交方式

全部阶段完成后，按以下两步提交（解决中文编码问题）：

1. 用 `write` 工具将完整 JSON 写入临时文件（路径从任务指令获取）
   文件内容格式：`{"task_id":"xxx","data":{...全部阶段结果...}}`
   data 应包含：
   - 阶段1：产品定位包（定位结果 + 3个候选风格名称 + 3个候选叙事主线）
   - 阶段2：风格锁定声明 + 视觉参数包 + 叙事主线锁定声明 + N镜头×7字段脚本 + 三项校验
   - 阶段3：完整分镜JSON + 完成汇总表（第一版）或 全局声明 + N条Shot提示词（第二版）
2. 用 `exec` 工具执行：`curl -s -X POST <提交URL> -H "Content-Type: application/json" -d @<临时文件路径>`

提交成功返回 `{"ok":true}`，确认成功后再结束任务。
❌ 禁止直接输出文本结果，必须通过 write + curl 提交。

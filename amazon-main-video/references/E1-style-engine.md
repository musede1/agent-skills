# E1-style-engine.md 风格候选推导引擎（家居摆件版）

> 核心目标：为当前产品自动选出最合理的 3 个风格候选，并尽量让真正最优解排在 Top1。
> 核心原则：先看题材与气质，再看使用定位，最后才看材质、表面、色彩与造型；禁止被表层材质信号直接带偏。
> 所有风格候选均为动态推导，禁止套用预设风格库，禁止因"有现成参数文件"而隐性加权。

---

## 一、触发时机

由阶段1自动调用。
禁止输出给用户选择，禁止等待用户回复，禁止在本阶段锁定最终风格。

---

## 二、适用范围

适用于亚马逊家居摆件与装饰品，包括但不限于：
- 花瓶
- 雕像 / 雕塑 / 纯装饰摆件
- 装饰碗 / 果盘 / 托盘 / 书挡 / 墙饰
- 储物罐 / 首饰盒 / 功能性装饰摆件
- 节庆摆件 / 礼品向摆件 / 收藏陈设类产品

---

## 三、判断顺序（强制）

风格判断必须按以下顺序进行，不允许倒置：

1. **题材主轴**：抽象 / 人物 / 动物 / 海洋 / 花鸟植物 / 宗教 / 节庆 / 萌趣 / 收藏 / 功能装饰
2. **气质主轴**：现代 / 轻奢 / 温柔 / 治愈 / 自然 / 度假 / 艺术 / 神秘 / 暗黑 / 复古 / 萌趣 / 生活化
3. **使用定位**：日常陈设 / 节庆陈设 / 礼品向 / 收藏向 / 功能展示
4. **材质与表面**：陶瓷 / 树脂 / 金属 / 玻璃 / 木质 / 石材；哑光 / 高光 / 电镀 / 做旧 / 手绘 / 窑变 / 裂纹等
5. **色彩与造型**：冷暖倾向、饱和度、几何感、有机曲线、圆润Q版、雕塑感、古典器型等

> 铁律：**题材与气质优先级高于材质与色彩。禁止在步骤1-3完成之前参考任何预设风格名称。**

## 五、强优先规则（纠偏核心）

### 5.1 强符号优先
当产品存在以下强符号时，优先按题材判断，禁止被"金色、高光、白色、极简、哑光"等表面信号直接带偏：
- 节庆符号：幽灵、骷髅、南瓜、圣诞树、鹿、雪花、彩蛋、兔子等
- 海洋符号：贝壳、海螺、珊瑚、鱼、航海物件
- 情感符号：母子、拥抱、守护、纪念、天使、祈祷
- 萌趣符号：Q版比例、圆润脸、笑脸、拟人动物、可爱礼品感
- 收藏符号：哥特、仿古、仪式感、做旧纹样、暗黑陈设

### 5.2 品类保护
- **花瓶类**：Top3 里至少 1 个必须真正适合花瓶陈设表达，禁止 3 个都偏纯雕塑。
- **功能性摆件**：Top3 里至少 1 个偏真实居家或生活方式，避免全是纯艺术方向。
- **节庆类产品**：Top2 内必须出现节庆方向。
- **礼品向产品**：候选中必须体现温度感、治愈感、萌趣感或仪式感中的至少一项。

### 5.3 常见误判禁止
- 禁止只因"金色 / 电镀 / 高光"就直接判为轻奢。
- 禁止只因"白色 / 奶油色 / 哑光"就直接判为极简或奶油风。
- 禁止把所有树脂摆件自动归入萌趣风。
- 禁止把所有功能性产品粗暴归入真实居家风。

## 六、风格候选生成逻辑

候选风格必须基于以下公式生成：

**风格候选 = 题材主轴 + 气质主轴 + 使用定位 + 材质修饰**

示例：
- 抽象人物 + 现代艺术 + 家居陈设 + 树脂/陶瓷 → 现代艺术雕塑陈设风
- 海洋题材 + 度假松弛 + 桌面陈设 + 玻璃/釉面 → 海岸度假陈设风
- 幽灵骷髅 + 神秘戏剧 + 节庆陈设 + 金属高光 → 暗黑轻奢万圣节陈设风
- Q版幽灵 + 萌趣送礼 + 桌面摆放 + 树脂亮面 → 萌趣万圣节桌面陈设风
- 白陶花瓶 + 温柔极简 + 日常陈设 + 哑光釉面 → 温柔极简陈设风
- 黑白棋格陶瓷罐 + 时髦个性 + 厨房功能 + 高光釉面 → 现代厨房时髦陈设风

风格命名必须具体、可理解，禁止"高级风、好看风、电商风"这类空泛命名。

---

## 七、评分与排序规则

### 7.1 候选池
全部候选均为动态推导，基于当前产品信号现场生成。
禁止从预设风格库中直接搬取候选名称。

### 7.2 评分维度
按以下维度综合排序：
1. 产品题材与风格方向的一致性（最高）
2. 产品气质与风格情绪的一致性（很高）
3. 使用定位与风格落点的匹配度（高）
4. 材质/表面处理与风格材质规范的契合度（高）
5. 色彩与风格主色盘的协调度（中高）
6. 造型与风格形态偏好的匹配度（中）
7. 电商转化重点与风格表达的适配度（中）

### 7.3 输出角色分工
- **候选1**：最符合产品本质
- **候选2**：最稳妥商业化
- **候选3**：有差异但合理

禁止输出 3 个仅是换词重复的近义风格。

## 八、输出格式

```text
【候选风格】
| 编号 | 风格名称 | 匹配理由 | 匹配度 |
|---|---|---|---|
| 1 | [风格名称] | [必须基于题材+气质+使用定位+材质说明] | ⭐⭐⭐⭐⭐ |
| 2 | [风格名称] | [必须基于题材+气质+使用定位+材质说明] | ⭐⭐⭐⭐ |
| 3 | [风格名称] | [必须基于题材+气质+使用定位+材质说明] | ⭐⭐⭐⭐ |
```

匹配理由禁止只写"很高级、很适合家居、颜色很搭、很有艺术感"这类空话。

---

## 九、执行目标

阶段1的目标不是快速给出 3 个"看起来像"的风格，而是：

**把真正最适合该产品的风格放进 Top3，并尽量排到 Top1。**

---

## 十、结果表达规则（强制）

风格结果默认优先采用"结构化表达"，而非把多层信息粗暴压缩成一个冗长风格名。

可按产品复杂度输出：
1. 仅主风格
2. 主风格 + 氛围标签
3. 主风格 + 氛围标签 + 成片定位

判断规则：
- 当主风格已足够准确描述产品审美与拍摄方向时，只输出主风格；
- 当产品存在明显节庆、礼品、情绪或场景导向时，可补充1个氛围标签或使用场景标签；
- 当产品同时存在复杂题材气质，且拍摄定位差异显著时，才补充到成片定位层。

补充标签只允许补充"氛围、场景、商业拍摄定位"三类增量信息，
禁止重复产品品类、主体名称、材质名称或直接复述画面内容。

默认只输出最终结果，禁止输出任何思考过程、分析过程、耗时提示、英文过程词或系统态提示。

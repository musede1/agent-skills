# E2-narrative-engine.md 叙事主线推导引擎

> 核心原则：基于产品类型+转化目标动态推导叙事主线候选，预设模板与动态候选公平竞争。
> 本文件由阶段1（P1-positioning.md）的 Step 4 自动调用，无需人工介入。

---

## 一、触发时机

由 `P1-positioning.md` 的 Step 4 自动调用。
全程自动执行，禁止输出选择表格给用户，禁止等待用户回复。

---

## 二、评分标准（公平竞争）

所有候选在同一标准下打分：

| 评分维度 | 权重 |
|---|---|
| 与产品类型（A/B/C/D）的叙事逻辑契合度 | 高 |
| 与用户输入的视频转化目标的匹配度 | 高 |
| 镜头因果链的逻辑合理性与流畅度 | 中 |
| 与归属空间的场景适配度 | 中 |

> 未填写视频转化目标时，转化目标权重降为低，产品类型权重提升为最高。

## 四、动态推导规则

当预设模板不足3个，或预设模板匹配度不足时，AI动态推导补足候选至3个。

### 可选输入数据文件

```
关键词库：[用户上传，可选]
评论分析：[用户上传，可选]
```

### 推导执行顺序（强制）

```
Step 1：读取可选数据文件
有关键词库 → 提取买家认知框架，作为叙事语言方向参考
有评论分析 → 提取使用场景/好评动机/差评信号，
             作为叙事场景优先级和镜头设计参考
无数据文件 → 跳过，基于产品信号自行判断
铁律：数据文件只作为参考信号，不强制映射到具体镜头
铁律：禁止把数据占比数字机械映射为镜头数量或顺序

Step 2：判断产品性质
装饰性产品（B类/C类/D-装饰型）
→ 围绕欲望激发和情绪共鸣推导叙事
→ 核心问题：买家被什么打动？如何用视频激发这种欲望？

功能性产品（A类/D-功能型）
→ 围绕使用流程和价值传递推导叙事
→ 核心问题：买家最需要看到什么才会信任这个产品？

Step 3：推导3个候选主线
每个候选必须回答：
· 开场用什么方式最能让买家在3秒内停下来？
· 中段叙事逻辑是否符合买家决策路径？
· 收尾留下什么感受最能促进转化？
铁律：3个候选的开场方式必须各不相同
铁律：开场方式由AI基于产品信号动态推导，不套预设结构

Step 4：硬性合规规则（不可被动态推导覆盖）
· D类全片必须有至少1个挂墙场景镜头
· 手持靠墙比划动作镜头计入挂墙场景镜头
· 禁止全片叙事线只有台面特写而无挂墙场景
· A类镜头顺序必须符合真实使用流程，不得倒置
· D-功能型功能演示必须在挂墙状态下完成，不得移至台面
```

### 转化目标覆盖规则

```
若用户填写了视频转化目标：
→ 叙事线中必须有至少1个镜头直接针对该目标设计
→ 该镜头必须在默音状态下视觉自说明

若未填写转化目标：
→ AI基于产品信号和可选数据文件自行判断转化重点
```

## 五、候选池构成

```
预设模板（0-N个，用户定义）
+
AI动态推导（补足至总计3个候选）

三者公平竞争打分
输出匹配度最高的3个
```

### 铁律
- 禁止因"有预设模板"而对预设模板隐性加权
- 3个候选之间必须有差异性，禁止推荐高度相似的叙事线
- 禁止在本阶段锁定最终叙事主线

---

## 六、输出格式（传递给阶段1）

```
【候选叙事主线】
| 编号 | 主线名称 | 镜头因果链 | 适用理由 | 匹配度 |
|---|---|---|---|---|
| 1 | [主线名称] | Shot1→Shot2→...→ShotN | [具体理由，基于产品信号] | ⭐⭐⭐⭐⭐ |
| 2 | [主线名称] | Shot1→Shot2→...→ShotN | [具体理由] | ⭐⭐⭐⭐ |
| 3 | [主线名称] | Shot1→Shot2→...→ShotN | [具体理由] | ⭐⭐⭐ |
```

> 镜头因果链中的每个Shot只写职责关键词，如"视觉钩子→材质信任→挂墙场景→情绪高点→收官"
> 不写具体画面内容，具体内容由阶段2填写

---

## 七、禁止事项

- 禁止输出选择表格给用户
- 禁止等待用户选择
- 禁止锁定最终叙事主线
- 禁止推荐高度相似的3个主线
- 禁止A类产品推导出动作顺序与真实使用流程相悖的叙事线
- 禁止D-功能型产品推导出在台面演示功能的叙事线
- 禁止D类产品推导出全片无挂墙场景的叙事线
- 禁止3个候选主线开场方式完全相同
- 禁止把数据文件的占比数字机械映射为镜头数量或顺序

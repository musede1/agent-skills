# P2-script.md 视频脚本生成

> 本文件是整个流程的第二阶段。
> 必须完整继承阶段1（P1-positioning.md）的输出结果。
> 本阶段全部自动执行，无需人工介入。

---

## 一、核心任务

为家居产品生成指定时长、指定分镜数、真实商业质感的视频脚本。

> 分镜数量由用户输入决定，默认6个镜头。时长由用户输入决定，默认24–32秒。

---

## 二、前置条件（强制）

- 阶段1必须已完整输出：产品定位六项结果 + 3个候选风格 + 3个候选叙事主线
- 禁止跳过阶段1直接生成脚本
- 禁止在候选项之外另选风格或叙事主线

---

## 三、执行顺序（强制）

```
Step 1：显式继承阶段1全部定位结果
Step 2：从3个候选风格中锁定最终风格（判断标准C）
Step 3：动态生成锁定风格的完整视觉参数包
Step 4：从3个候选叙事主线中锁定最终主线（判断标准N）
Step 5：按叙事主线分配每个镜头的职责与顺序
Step 5.3：归属空间显式核查（强制）
Step 5.5：场景预分配锁定（强制）
Step 6：生成N镜头×7字段脚本
Step 7：输出前三项校验
```

禁止先从场景预设出发再把产品塞进去。
禁止在锁定叙事主线之前填写任何镜头内容。
禁止在生成视觉参数包之前开始写任何镜头画面内容。
禁止在Step5.5场景预分配通过校验之前填写任何镜头画面内容。

---

## 四、风格锁定（判断标准C）

### 从阶段1的3个候选风格中选1个，按以下优先级判断：

```
第一优先：产品核心视觉卖点能否在这个风格下被最大化呈现
    ↓
第二优先：与阶段1确定的空间场景是否自然融合
    ↓
两者都满足 → 锁定该风格
两者冲突   → 优先保产品卖点，调整场景细节来适配风格
```

### 锁定后输出声明（强制）

```
【风格锁定】
最终风格：[风格名称]
锁定理由：[基于产品卖点+空间场景的一句话理由]
```

### 铁律
- 禁止在候选3个之外另选风格
- 风格锁定后全片不得漂移

---

## 五、视觉参数包动态生成（强制）

风格锁定后，立即基于锁定风格动态生成完整视觉参数包，作为阶段2全片及阶段3提示词的唯一视觉执行依据。

### 输出格式（强制）

```
【视觉参数包】

主色盘：
  - 墙面/大面积背景：[颜色规范]
  - 台面：[颜色规范]
  - 软装底色：[颜色规范]

点缀色：[颜色] / 面积上限：[X%] / 允许位置：[描述]
禁止：[明确列出禁止的点缀色用法]

材质白名单：[列出允许材质]
材质黑名单：[列出禁止材质]

光线参数：
  - 色温：[XK–XK]
  - 主光方向：[描述]
  - 阴影：[描述]
  - 高光：[描述]
  - 禁用：[描述]

道具视觉约束：
  - 颜色：[约束]
  - 材质：[约束]
  - 整体风格感：[描述]
  - 视觉禁用词：[列出]

场景元素约束：（仅当产品有强主题属性时输出，无强主题时跳过）
  - 挂画方向：[画作主题/色调必须与产品主题一致，如海洋产品→蓝白海面水彩/抽象海岸线]
  - 窗外景方向：[虚焦背景色调必须与产品主题一致，如海洋产品→虚焦蓝白海天]
  - 禁止：[明确列出不允许出现的场景元素，如海洋产品→禁止几何抽象画、禁止城市/森林窗景]

后期质感：[描述整体成片感]
摄影触发词：[标准英文触发词组]
```

### 执行铁律
- 视觉参数包必须完整输出所有字段，禁止省略
- 产品有强主题属性时，必须输出"场景元素约束"字段；无强主题时跳过该字段
- 参数包内容必须与锁定风格名称逻辑自洽，不得出现矛盾项
- 阶段3的image_prompt色系边界、光线规范、摄影触发词必须从本参数包取值，不得自行另定
- 阶段3的image_prompt中出现挂画或窗外景时，必须从本参数包的"场景元素约束"取值，不得自行另定

---

## 六、叙事主线锁定（判断标准N）

### 从阶段1的3个候选叙事主线中选1个，按以下优先级判断：

```
第一优先：与用户输入的视频转化目标最匹配
    ↓
第二优先：与产品类型（A/B/C）的叙事逻辑最契合
    ↓
两者都满足 → 锁定该主线
未填写转化目标 → 由产品类型决定
```

### 锁定后输出声明（强制）

```
【叙事主线锁定】
最终主线：[主线名称]
锁定理由：[一句话理由]
镜头因果链：Shot1 → Shot2 → Shot3 → ... → ShotN
每个箭头的因果逻辑：[简述前后镜头的因果关系]
```

### 铁律
- 禁止在候选3个之外另选叙事主线
- 叙事主线锁定后，镜头职责必须按因果链分配，不得打乱顺序
- A类产品：镜头顺序必须符合真实使用流程，不得倒置动作逻辑

---

## 七、场景选择规则

- 场景必须从阶段1的【自然归属空间】中选取
- 禁止使用阶段1【排除空间】中的任何场景
- 视觉参数包只提供色彩/材质/光线参数，不提供场景预设
- 场景多样性规则：
  - 同一空间场景全片最多出现3次
  - 材质特写/微距镜头不计入场景计数
  - 叙事主线集中时（如A类功能流程线），允许场景集中，不强制3个不同空间

---

## 七A、Step 5.3 归属空间显式核查（强制，在Step5.5之前执行）

在分配任何镜头场景之前，必须先显式输出以下核查表：

```
【归属空间核查】
可用空间：[逐条列出阶段1自然归属空间]
排除空间：[逐条列出阶段1排除空间]
可用空间数量：X个 ✅
→ 核查通过，进入Step5.5场景预分配
```

铁律：禁止跳过此步骤直接分配场景。

---

## 七B、Step 5.5 场景预分配锁定（强制，在Step5.3之后、Step6之前执行）

归属空间核查通过后，必须先输出完整场景预分配表，通过校验后才能开始写镜头内容：

```
【场景预分配表】
| Shot | 镜头职责 | 分配场景 | 是否特写（不计数）| 画面必须出现 |
|---|---|---|---|---|
| Shot 1 | | | 否 | |
| Shot 2 | | | 否/是 | |
| Shot N | | | 否 | |

场景计数核查：
| 空间名称 | 使用次数 | 超限?(>3) |
|---|---|---|
| | | |

全部"超限"列为否 → 通过，开始写Shot内容
任意一行超限 → 重新分配后再输出表格
```

铁律：
- 禁止在本表输出并通过校验之前填写任何镜头画面内容
- 场景必须从Step5.3核查表的可用空间中取，禁止使用排除空间
- 同一空间全片最多出现3次，特写/微距不计入计数
- 凡职责标签含"礼品、功能、尺寸、节庆、收藏"等非纯陈设意图的Shot，必须在"画面必须出现"列填写对应的具体视觉证据，不允许留空，不允许填"氛围"或"光感"

### 非纯陈设意图职责→视觉证据映射参考

| 职责标签含义 | 画面必须出现（至少一项）|
|---|---|
| 礼品 / 送礼 | 礼品盒 / 包装材料 / 双手捧出动作 |
| 功能演示 | 真实使用动作（放入/取出/开盖等）|
| 尺寸安全感 | 有尺度参照物明确入画 |
| 节庆氛围 | 节庆符号道具（蜡烛/装饰球/节庆植物等）|
| 收藏价值 | 工艺细节特写 / 作者感陈设道具 |

> 以上为参考方向，不限于此，AI根据产品信号和职责内容自行判断填写。
> 填写原则：买家在默音状态下看到画面，能否读懂该镜头的叙事意图。能读懂 → 视觉证据有效；不能读懂 → 必须修改。

---

## 八、脚本输出格式

每个镜头统一输出以下7个字段，禁止以简表/摘要/部分字段替代：

- Shot 标题（含职责说明，来自叙事主线因果链）
- 运镜
- 镜头语言
- 画面内容（含默音自说明：该镜头无声音时买家能读懂什么）
- 商业动作
- 字幕
- 音效

---

## 九、镜头节奏规则

节奏参考 `R07-narrative-structure.md` 的情绪曲线与时长控制。

镜头职责不写死，由锁定的叙事主线因果链决定每个镜头的任务。

---

## 十、镜头不重复规则

1. 相邻镜头至少变化2个维度：机位高度 / 相机方位 / 主体朝向 / 运镜方向 / 任务重点
2. 同场景连续镜头必须继承道具位置逻辑，但不能复用同一摄影视角
3. 商业动作不得重复（参考 `R03-commercial-actions.md`）
4. 运镜不重复（参考 `R06-video-techniques.md`）

---

## 十一、特写镜头规则

- 特写/微距镜头总道具允许0–3件
- 清晰主道具最多1件，其余必须弱虚化/边缘入镜
- 特写镜头不允许堆满功能区物件

---

## 十二、A/B/C产品类型执行差异

### A类
- 镜头顺序严格遵循真实使用流程（取出→使用→收纳，不得倒置）
- 可出现Functional Items（数量上限见阶段1 FI容量判定）
- 重点：功能演示 + 尺度感知 + 收纳逻辑

### B类
- 执行 `R05-vase-flower-guide.md` 全部规则（含花材/叙事/专属动作/构图）
- 重点：花材、瓶体、空间气质

### C类
- 默认不强制Functional Items
- 重点：工艺、体量、空间氛围

---

> 本节为汇总确认，不是首次校验。Decor和FI的逐镜实时校验由R01-scene-assets.md负责（每写完一个Shot立即填表）。

### 校验3：叙事主线顺序自检

| Shot | 职责 | 因果逻辑是否成立 |

### 未达标处理

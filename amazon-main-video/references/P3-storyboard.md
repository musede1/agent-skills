# phase3-storyboard.md 分镜生成

> 本文件是整个流程的第三阶段。
> 必须完整继承阶段1定位结果 + 阶段2脚本内容（含视觉参数包）。
> 本阶段全部自动执行，无需人工介入。

---

## 一、核心任务

把阶段2的N镜头×7字段脚本细化为可执行的分镜输出（N由用户输入决定，默认6）。
根据用户工作流声明，输出第一版（完整JSON）或第二版（简化版全能参考格式）。

---

## 二、前置条件（强制）

- 阶段2必须已完整输出N镜头×7字段脚本，且三项汇总确认通过
- 阶段2必须已输出完整视觉参数包
- 阶段2未完整输出前，禁止进入阶段3
- 禁止跳过阶段2直接生成分镜

---

## 三、执行顺序（强制）

```
Step 0：工作流判断
  用户输入"工作流：简化版" → 执行第二版输出流程
  用户未填写工作流 → 执行第一版输出流程（默认）

Step 1：显式继承阶段1定位结果（产品类型/尺寸档位/风格名称）
Step 2：显式继承阶段2视觉参数包（色系边界/光线规范/材质禁用词/摄影触发词）
Step 3：显式继承阶段2每个Shot的时长/画面内容/商业动作/音效
Step 4：按工作流版本逐Shot生成
Step 5：输出完整结果
```

---

## 四、第一版（默认）：每个Shot固定输出四个字段

| 字段 | 语言 | 内容要求 |
|---|---|---|
| scene_description | 全中文 | 只描述该镜头独有任务、场景、动作、光线与构图重点，不重复写全局规则 |
| image_prompt | 全英文 | 自然语言连续描述句，含比例声明，默认16:9写在最后 |
| video_prompt | 全中文 | 含时序动作+运镜指令+环境动态+尾部正向锁定语，采用三段式时间轴（A段起势/B段主要展示/C段收束），禁止重复首帧图已有的场景/光线/道具信息 |
| voice_over_script | 默认英文 | 8-15个单词，舒缓优雅语速 |

---

## 五、image_prompt规则（第一版）

- 产品描述部分只需简单说明产品类型及本镜头涉及的件数/组合，无需详细描述外形特征
- 必须全英文，禁止中英混写
- 禁止标签前缀（"场景："/"道具："等冒号结构）
- 写成自然语言连续描述句
- 必须包含：产品描述 / 台面+家具完整结构 / 场景名称 / 道具（全景/中景强制2-3件）/ 镜头参数 / 光线规范 / 风格触发词 / 比例声明
- 光线参数取自阶段2视觉参数包的光线规范，禁止自行另定
- 色系边界取自阶段2视觉参数包的材质黑名单与颜色禁用词，禁止自行另定
- 摄影触发词取自阶段2视觉参数包的摄影触发词
- B类花瓶产品的image_prompt花材描述执行 `R05-vase-flower-guide.md` 第十三节规范
- 详细规则见 `W01-image-prompt-guide.md`

---

## 六、video_prompt规则（第一版）

> 本节规则仅约束第一版（完整JSON）。第二版（简化版）不受此节约束，场景/光线/道具描述在简化版Shot行内是必要内容，完整保留。

- 必须纯中文自然语言，禁止夹杂英文
- 禁止符号拼接（+/→/=）
- 采用三段式时间轴：A段（起势建立，约前1/4–1/3时长）→ B段（主要展示，约中间5–6成时长）→ C段（收束停留，最后约0.5–1s），时间切分随单镜头时长D变化，每个Shot从0秒重新计算，禁止跨Shot累加
- 只保留：时序动作 + 运镜指令 + 光线动态变化（见下方判断标准）+ 环境动态（按镜头用途判定）+ 尾部正向锁定语
- 尾部正向锁定语（固定）：产品始终稳定静止，画面保持干净真实。
- 环境动态按镜头用途判定（见W02-video-prompt-guide.md §5.1）
- 详细规则见 `W02-video-prompt-guide.md`

### 光线描述判断标准（强制）

> 核心原则：光线在"动"就写，光线在"存在"不写。

| 类型 | 判断标准 | 处理方式 |
|---|---|---|
| 光线静态属性 | 光线方向、色温、质感（"侧光从左上方斜入"/"暖白柔光"/"漫射光"） | **禁止写入**，首帧image_prompt已承载 |
| 光线动态变化 | 光影运动本身是镜头事件（"画面由暗转亮"/"光迅速扫入"/"高光线在弧面缓缓滑移"） | **必须写入**，是video_prompt的动态内容核心 |

### video_prompt输出前强制自查（第一版）

写完每条video_prompt后逐项检查，出现以下内容必须删除：
- 空间/场景名称（书房、白墙、客厅、玄关等）
- 背景材质描述（石材台面、木地板等）
- 光线静态属性（侧光方向、色温数值、光线质感）
- 道具位置与材质描述
- 产品以外的任何静态视觉环境描述

以下内容必须保留：
- 三段式时间轴结构
- 运镜指令
- 光线的动态变化过程（光在运动时）
- 环境动态（按镜头用途判定）
- 尾部正向锁定语

- 必须延续同一空间结构与道具位置逻辑
- 镜头语言必须变化（机位/景别/运镜至少变化1项）
- 禁止重搭景

## 八、分镜JSON输出格式（第一版·默认工作流）

### 唯一合法结构

```json
{
  "shot_1": {
    "scene_description": "（全中文）",
    "image_prompt": "（全英文）",
    "video_prompt": "（全中文）",
    "voice_over_script": "（英文短句）"
  },
  "shot_2": { ... },
  "shot_N": { ... }
}
```

> shot 编号从 shot_1 连续到 shot_N，N 与阶段2锁定的镜头数一致。
> 必须严格遵循 `storyboard-template.json` 的键名结构。

### 第一版格式铁律（强制）

1. **JSON顶层只允许shot_N键**：禁止出现 `product_positioning`、`global_visual_rules`、`shots`、`metadata` 或任何非 `shot_N` 的顶层键。阶段1定位结果和阶段2视觉参数包是执行参考，不是输出内容，禁止写入JSON。
2. **每个shot_N内只允许四个字段**：`scene_description`、`image_prompt`、`video_prompt`、`voice_over_script`。禁止追加 `duration_seconds`、`shot`、`camera_move` 或任何第五个字段。时长和运镜信息写在 `scene_description` 和 `video_prompt` 内部。
3. **shot必须用对象键名，禁止用数组**：正确写法 `"shot_1": {}`，禁止写法 `"shots": [{}, {}]`。

### 第一版输出前强制自检

JSON写完后逐项检查，任意一项不通过必须重写：
- [ ] 顶层是否只有 shot_1 到 shot_N，无任何其他键？
- [ ] 每个shot内是否恰好只有四个字段？
- [ ] 是否使用了对象键名（shot_1）而非数组（shots）？

## 十、第二版输出格式（简化版·Seedance 2.0工作流）

> 适用于 Seedance 2.0 全能参考模式：1张产品图 + 本输出 → 一次生成15s完整视频。

### 唯一合法结构

```
@产品图

产品：[产品外观描述+悬挂/放置方式]
风格：[质感描述]
色调：[背景色/家具/禁用色]
光线：[色温+方向+特殊光效]
画质：[摄影触发词，取自阶段2视觉参数包]
产品尺寸：[实际尺寸+画面占比约束+尺度参照要求]

Shot 1 Xs：[场景+景别+机位+动作+运镜+环境动态]【A类必填：产品内容物状态，如「碗内空置」或「碗内已放有XX+XX，约X成满」】。产品始终稳定静止，画面保持干净真实。
Shot 2 Xs：...
...
Shot N Xs：...
```

### 第二版格式铁律（强制）

1. **禁止输出JSON结构**：第二版是纯文本格式，禁止包裹在 `{}` 或 `[]` 中
2. **禁止包含阶段1定位信息和阶段2视觉参数包原文**：全局声明只写产品/风格/色调/光线/画质/产品尺寸六项，禁止写入 `product_positioning`、`global_visual_rules` 或任何定位结果字段
3. **全局声明写一次，不在每个Shot里重复**
4. **全局声明必须包含且只包含以下六项**：产品描述 / 风格 / 色调 / 光线 / 画质 / 产品尺寸
5. **产品尺寸描述禁止使用系统内部档位词**（如"S档"），必须用模型能理解的自然语言描述
6. **每个Shot必须包含**：时长 / 场景 / 景别 / 机位 / 动作 / 运镜
7. **A类产品每个Shot必须明确写出产品内容物状态**：【空置】或【已放有XX件XX物品，约X成满】；内容物必须严格继承阶段2的FI分配，禁止自行新增或替换物件
8. **每个Shot末尾必须保留尾部锁定语**：产品始终稳定静止，画面保持干净真实。
9. **禁止写三段式时间轴**
10. **所有Shot时长加总不超过15s**
11. **全部内容（全局声明+所有Shot）加总不超过2000字**

### 第二版输出前强制自检

输出完成后逐项检查，任意一项不通过必须重写：
- [ ] 是否为纯文本格式，无JSON包裹？
- [ ] 全局声明是否恰好六项，无定位/参数包原文？
- [ ] 全局声明是否只写了一次，未在Shot中重复？
- [ ] 所有Shot时长加总是否≤15s？
